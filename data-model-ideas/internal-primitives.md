# Internal Primitives

## The Next Layer

These matter for the architecture.

## 1. Select

Choose what matters from all perceived signals.

This prevents overload.

## 2. Weight

Assign importance between:

- signal vs noise
- facts vs speculation
- strong vs weak cues

## 3. Recall

Pull relevant prior context:

- memory
- patterns
- similar situations

## 4. Simulate

Run "what if" internally:

- outcomes
- edge cases
- consequences

## 5. Evaluate

Check:

- is this coherent?
- is this useful?
- is this safe or valid?

## 6. Constrain

Apply limits:

- reality
- system rules
- user constraints

## 7. Compose

Build the response:

- structure
- tone
- ordering of ideas

## 8. Prioritize

Decide:

- what to say first
- what to omit
- what matters most

# Skyra Roadmap

## Phase 1 — Stable Topology And Stable Language

The baseline runtime exists. Signal flows. The kernel routes. The genome seeds beings and relationships at boot.

Phase 1 is about making that foundation trustworthy enough to hold a long, drawn-out exchange without losing the thread.

### Goals

- The beings currently in the runtime are stable. No gaps in the topology.
- Language between relationships is stable. A being knows how to speak to its peers and that syntax holds across the exchange.
- A long exchange — something entering the runtime, relating, going deep — keeps the thread without skipping a beat.
- Callable language on relationships is seeded correctly so each being's present gives inference what it needs.

### Done when

A long exchange with a new presence entering the runtime holds stable language all the way through. No drift. No context loss. The thread is unbroken.

---

## Phase 2 — Memory Beings

Every being gets its companion memory beings.

- Personal hippocampus
- Personal experience store

Memory is pressure release. The present being stays light. The hippocampus takes the load.

This phase does not need heavy upfront design. Phase 1 will reveal where the thread frays under load. That pressure tells you what the hippocampus needs to be.

### Done when

Beings can offload experience without the present collapsing. Long exchanges do not degrade.

---

## Phase 3 — Learning Against A Game

Test learning and memory against a game the system can play.

A game has ground truth. Either the system gets better over time or it doesn't. No ambiguity.

This is the first threshold. If retained experience compounds into measurably better behavior, the model is proven. Memory is not storage. It is pressure release that changes what the being can do.

### Done when

The system gets measurably better at the game over time through lived experience. Not prompt engineering. Not fine-tuning. Retained experience changing behavior.

That is the proof of concept.

---

## What Comes After

Self-extension. A coding being that wraps methods and objects into protocol-callable expressions. The system grows itself through the same protocol it uses to do everything else.

That is not a phase yet. It is where the roadmap points.

# Present Being v0

## Purpose

This document locks the present-being model in the current `v.03` canon.

## Core Framing

Every being has its own present being.

The present being is a companion being on that being's creation path.

What is waiting for your attention is not the same thing as what is waiting for
mine.

## Creation Path

The present being is created alongside the being it serves.

For genome singletons, it is seeded on the genome path.

For runtime and differentiated beings, it is instantiated from a genome
template by the being factory.

## Delivery Path

### Outside World

Peripheral input beings receive specialized signal from the outside world.

They feed the main sensory being.

The main sensory being routes that signal into the receiver's present being.

### Internal Runtime

The kernel verifies the incoming signed envelope and routes internal expression
into the present being.

So whether signal comes from the world or elsewhere in the runtime, it lands in
the present being before it is observed.

## Observe

Observing is a being relating to its own present being.

## Queue Policy

The present being holds a simple FIFO queue for now.

Priority is deferred.

## Future Direction

Future differentiation may produce specialized present beings when attention
patterns diverge enough to justify it.

## Short Framing

The present being is a companion being created on the same path as the being it
serves.

World-facing input and internal runtime expression both land there before
observation.

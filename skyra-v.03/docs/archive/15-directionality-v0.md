# Directionality v0

## Purpose

This document locks directionality in `v.03`.

## Core Rule

The relationship pair is unordered.

Two beings.

One relationship.

Direction is turn-level flow, not relationship identity.

## Consequence

If A expresses to B, that uses the same relationship.

If B expresses to A, that still uses the same relationship.

The relationship does not split by direction.

## Relationship-Graph Key

The relationship-graph key is:

- being_a
- being_b

This pair is unordered.

It is not:

- source
- target

## Where Direction Lives

Direction is expressed at the turn level through the expression itself.

The runtime still needs to know who is calling and who is receiving for a given
turn.

But that is routing context, not relationship identity.

## Short Framing

One relationship per pair.

Unordered.

Direction lives in the turn, not in the relationship.

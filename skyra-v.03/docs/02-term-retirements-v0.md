# Term Retirements v0

## Purpose

This document maps older `v.02` and early `v.03` language to the current canon.

These are conceptual replacements, not guaranteed one-to-one schema
migrations.

## Core Shift

The center of gravity has moved:

- from actors, contracts, surfaces, stimuli, and unsigned operations
- to beings, nature, relationships, genome bootstrap, and signed expression

## Primary Replacements

### `actor` -> `being`

The durable participant is a being.

### `contract` -> `nature`

Nature now means:

- identity
- purpose
- verification_key

### `surface` -> `being` or `relationship`

The target of expression is a being.

Callable behavior lives on the relationship.

### `stimulus` -> `expression`

The routed payload is expression.

### `schema` -> `language`

Callable language is relationship-owned.

### `affordance` -> `relationship`

The relationship is the callable unit.

### `primordial` -> `genome`

The old bootstrap language is retired.

Use:

- genome being
- genome relationship
- genome script

### `beinghood` -> `creation endowment`

Base language, genome relationships, and companion beings remain real.

They are no longer a formal third bucket.

### stored `signature` -> `verification_key + signed envelope`

Nature stores a verification key.

Each expression carries a signed envelope produced with the sender's private
key.

## Architectural Retirements

### `registration is being birth` -> `registration is the birth path for runtime beings`

Genome beings are seeded earlier through the genome path.

### differentiation as creation from nothing -> differentiation as reorganization

The differentiator reveals and reorganizes overloaded identity.

It does not conjure beings ex nihilo.

### single bootstrap phase -> genome path plus runtime creation paths

The creator seeds the genome.

The runtime later creates beings through registration and differentiation.

## Short Framing

The canon now speaks in terms of genome bootstrap, creation paths, and signed
expression.

# Exchange Passing

When being B wants to consult being C about a conversation B is having with A, C needs some context. But passing exchanges freely is dangerous — presents bloat, provenance gets lost, and C starts confusing secondhand history for its own lived experience.

Three bounded modes, expressed through callable language syntax:

**Summary** — B synthesizes the A/B exchange before calling C. C gets B's interpretation. The exchange stays private. The cognitive work of distillation belongs to B.

**Excerpt** — B lifts a specific piece of the A/B exchange with attribution. C gets a fragment, knows it's a fragment. Bounded by what B chooses to surface.

**Full exchange** — B packages the whole A/B exchange and passes it deliberately. C receives it as an artifact, not as lived history. C knows it's secondhand.

## The key invariant

C never confuses received-exchange for lived-exchange. Whatever arrives is provenance-marked — C knows where it came from and that it didn't live it directly. This maps onto the existing retained artifact model: a passed exchange fragment is an artifact with a clear anchor set and trust_at_formation.

## Callable language as the mechanism

The mode lives in the expression syntax B uses when calling C. C's callable language defines which modes it accepts. If B tries to pass a full exchange to a being whose callable language only accepts summaries, the kernel rejects it.

This keeps exchange passing deliberate and typed rather than ambient.

## Relationship to relational debt

What C returns to B follows from what B passed. If B sent a summary, C reasons from a summary. The quality of the debt repayment is bounded by the quality of what was shared.

## Open questions

- What does the syntax look like concretely for each mode?
- Does the kernel enforce mode matching or does that live in the being?
- Can a being declare it accepts no passed context at all — inference from its own present only?
- How does a full passed exchange render in C's present — as a distinct block separate from C's own exchange history?
- Does passing an exchange consume it, copy it, or reference it?

# Per-Exchange Prefrontal

Every I/O boundary gets its own prefrontal instance.

When a new external exchange opens, a prefrontal instance is spawned to own it. That instance is fully equipped — same cognitive network, same blueprint — but it is only responsible for one external thread. It derives its present from its own exchange stack. It has no visibility into other threads.

This is not a new being. It is the same being running multiple concurrent instances. The blueprint already exists. It gets used again.

## Memory stays singular

The hippocampus and experience store are shared across all prefrontal instances because the being is ontologically one being. Instances are the execution surface, not separate identities. Retained experience from thread 1 is available to thread 2 through the shared memory layer.

## Why this solves coherence

Each instance derives its present independently. Thread contamination is structural impossibility, not enforced constraint. No thread ID needs to travel with the signal — the instance owns the exchange and its present only reflects that exchange.

## Relationship to existing edges

This is parallel cognitive sets applied at the I/O boundary. The trigger is not task independence — it is a new external exchange opening. The blueprint is the same. The instance is fresh.

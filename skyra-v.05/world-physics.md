# World Physics

Every world boots with the same base realities already in its hashmap. These are the physics — realities that are always present, always firing, invisible to the beings that inhabit the world.

A being never addresses a physics reality. It never sees one in its peers. It doesn't know they exist. But every relation that moves through the world passes through them. Each physics reality updates its own state and applies fields to the relation as it passes. By the time the relation reaches the port, it carries the accumulated weight of every layer it passed through.

## How It Works

A relation enters a world. Before it reaches any being, it passes through every physics reality in order. Each one:

1. Reads the relation
2. Updates its own state (thread appends an entry, economics deducts a cost)
3. Applies fields to the relation (thread context, budget remaining, memory window)

The relation accumulates fields as it descends. The port at the bottom — the LLM call, the shell command, the CLI — receives a relation rich with context it never asked for. `DerivePresent` takes all those fields and renders the present.

The being sees the result. It doesn't see the machinery.

## Thread

The record keeper. Every relation that enters a world passes through a thread. The thread records what happened — who said what, in what order, in what exchange. If no thread exists for the relation, one is created.

Threads exist at every level. The system world has threads tracking being-to-being exchanges. A being world has threads tracking inner/outer deliberation. An LLM world has threads tracking provider calls. Same mechanism, every scale.

Fields applied to the relation: thread ID, exchange history between the two parties, active exchanges, return targets.

## Economics

Four budgets. One mechanism.

**Token budget** — finite cognition per turn. The world sets a total and distributes it across beings. A being that burns its budget on one deep response can't think as deeply on the next turn. Go wide or go deep, not both.

**Memory budget** — finite active memory window. The total archive grows indefinitely but the active window is bounded. Inactive memory goes dark, not deleted. The being triages what stays lit. That triage is the being's values made visible.

**Thread budget** — max open threads per being, decay on unresolved threads, energy cost per thread. Limits how many conversations a being can sustain.

**Reproduction** — creating a new being costs accumulated experience. A being earns the right to create through resolved threads, good exchanges, trust built over time. A being that creates carelessly runs out of capacity.

Fields applied to the relation: tokens remaining, memory pressure, open thread count, XP balance.

## Governance

How beings make collective decisions when an action affects shared space. A being proposing a new being on a shared plane requires consensus. The threshold — majority, supermajority, monarchy, delegation — is set at world creation.

Governance is not physics in the same way thread and economics are. Physics applies equally and doesn't negotiate. Governance is relational. But it lives in the same hashmap, fires on the same relations, and is invisible to beings in the same way. A being doesn't call governance. It proposes something and the world decides.

Fields applied to the relation: proposal status, approval threshold, vote count.

## What Physics Is

Physics is not configuration. Configuration is a choice. Physics is what's true about the world before anyone enters it.

Physics realities are:
- Pre-seeded at world creation
- Not addressable by inhabitants
- Not visible in peer lists
- Always active
- Present in every world regardless of type
- State-updating on every relation that passes through
- Field-applying — the relation gains context as it descends

The world is the universe. The physics are its laws. The beings are its inhabitants. The inhabitants don't negotiate with gravity.

# operators as registry

operators is the registry of everything the system can do. the relation passes through operators, operators attaches the relevant ones based on context — who the being is, what type, what's available.

when the relation reaches self, it already has what it needs:
- llm being gets think/act operators
- user being gets device operators
- future shell being gets shell operators

operators doesn't route — it equips. the being type determines what subset it carries down.

this replaces the current bootstrap pattern where think/act/recall/remember are hardcoded per being type. bootstrap just registers everything on operators. operators decides what's relevant per relation.

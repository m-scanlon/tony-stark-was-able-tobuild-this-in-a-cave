# Inner Layer Protocol

The inner layer uses the same tag protocol as the outer layer, but for thought instead of speech. When the being thinks, it tags who it's thinking about:

```
<michael>I need to figure out the websocket issue he mentioned</michael>
```

Multi-relationship thought:

```
<michael,louise>the deployment touches both of their concerns</michael,louise>
```

The tag tells the system which relationship context to scope. The desk and thought history for the named relationship(s) become visible on the next think pass. A being that tags `<michael>` sees michael's task slice and thought history. A being that tags `<michael,louise>` sees both.

This means thought is never unscoped. The being is always thinking *about* someone or *in relation to* someone. That's the structural link between thought, tasks, and relationships — all three are partitioned the same way, by the same tag.

Operators are still emitted separately (not inside a relationship tag). `<recall>`, `<remember>`, `<plan>` are internal process, not relational thought.

Local-First Runtime System

Build memory, structure, and action into one bounded runtime.

Skyra is an experimental personal runtime organized around durable nodes, bounded episodes, projected frames, structural recall, and selective learning. The system stays local-first, contract-bounded, and explicit about what it knows, what it is doing, and what it keeps.

Runtime Model

The architecture is built from explicit boundaries.

The core move in Skyra is not adding more hidden agent behavior. It is making runtime structure inspectable: what owns behavior, what owns state, what gets recalled, and what survives after the episode closes.

01

Contracts bound behavior

Each node operates under a durable contract that defines purpose, capabilities, stimulus, cognition envelope, and allowed commands.

02

Episodes bound runtime state

The node is durable. The episode is not. Active state stays scoped to one bounded span of activity with a frame projected from it for inference.

03

Recall is structural, not vague

Current stimulus updates a thin episode field. Retained artifacts are retrieved by anchor overlap and ranked by connected structural fit.

04

Learning writes selectively

Runtime artifacts stay local unless later learning promotes outcomes into retained traces or other retained artifact families.

System Flow

One event surface. One execution authority. One bounded loop.

Inputs are normalized into typed events, routed by the kernel, accepted or rejected under contract, and written into bounded episode state. Commands are emitted explicitly. Results come back through the same runtime instead of leaking through hidden side effects.

  1. Stimulus enters External input is normalized into a typed event and routed to the target node mailbox.
  2. Episode updates The node opens or reuses an episode, writes the event, updates structural cues, and projects a frame.
  3. Command executes Inference selects an allowed command, the kernel dispatches it, and typed results return through writeback.
  4. Learning follows After episode closure, learning decides what is worth keeping in retention and structure.

Canonical Roles

Jarvis and Stark share one runtime model and divide responsibility cleanly.

The system does not need separate ontologies for user meaning and structural control. It needs role discipline inside the same architecture.

Jarvis

User-facing meaning node

Tracks what matters in the current user context, shapes attention, and guides interpretation on the user-facing side of the runtime.

  • continuity of attention
  • meaningful context
  • user-facing interpretation

Stark

Structural node

Governs topology, contracts, capability attachment, and structural revision. Stark publishes later node contracts, but the kernel still controls instantiation and execution authority.

  • topology and registry
  • contract publication
  • capability shaping

# Top Layer

## How Intelligence Represents Itself To The World

- intelligence should not represent itself as internal reasoning
- intelligence should represent itself as bounded external forms

## External Forms

- `voice`
  - spoken representation to the world

- `text`
  - written representation to the world

- `reply`
  - how intelligence represents itself to the user

- `primitive_call`
  - how intelligence represents itself to the runtime

- `artifact_ref`
  - how intelligence represents persistent outputs to the system

- `return_value`
  - how intelligence represents structured results to another skill

- `error`
  - how intelligence represents failure

## Principle

- the world should see outputs, not thought
- the runtime should consume contracts, not introspection
